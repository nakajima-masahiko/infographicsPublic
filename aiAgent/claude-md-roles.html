ディレクトリ構成例
my-app/ ├── CLAUDE.md # 全体ルール（命名規則・技術スタックなど） │ ├── backend/ │ ├── CLAUDE.md # バックエンド固有の指示 │ ├── src/ │ └── package.json │ ├── frontend/ │ ├── CLAUDE.md # フロントエンド固有の指示 │ ├── src/ │ └── package.json │ └── docs/ ├── architecture.md ├── api-spec.md └── db-schema.md
🌐

ルート CLAUDE.md

プロジェクト全体に適用される共通ルール。命名規則・コーディング規約・使用言語など横断的な設定を記述する。

⚙️

backend CLAUDE.md

Node.js / Express / DB 設計など、バックエンド専門の開発者として振る舞うための役割と制約を記述する。

🎨

frontend CLAUDE.md

React / TypeScript / UI コンポーネントなど、フロントエンド専門の開発者として振る舞うための役割と制約を記述する。

📚

参照ドキュメントの指定

CLAUDE.md から関連ドキュメント（API仕様書・DB スキーマなど）へのパスを記述し、自動的に参照させることができる。

CLAUDE.md の記述例
backend/CLAUDE.md
# バックエンド開発ルール ## あなたの役割 バックエンド専門の開発者として 振る舞ってください。 ## 技術スタック - Node.js + Express - PostgreSQL + Prisma ORM - Jest でテスト ## 守るべきルール - docs/api-spec.md の仕様に従う - 全エンドポイントにバリデーション - エラーは { error: { code, message } } - 1関数は50行以内 ## 参照ドキュメント - ../docs/architecture.md - ../docs/api-spec.md - ../docs/db-schema.md
frontend/CLAUDE.md
# フロントエンド開発ルール ## あなたの役割 フロントエンド専門の開発者として 振る舞ってください。 ## 技術スタック - React + TypeScript - Tailwind CSS - React Query でAPI通信 ## 守るべきルール - docs/frontend-spec.md の仕様に従う - コンポーネントは200行以内 - 型定義は backend/types/ を参照 ## 参照ドキュメント - ../docs/architecture.md - ../docs/frontend-spec.md
実行コマンド例
Terminal
# ── バックエンド作業 ──────────────────────────
$ cd backend
→ backend/CLAUDE.md を読み込み
→ "バックエンド専門開発者" として動作開始
$ claude "タスクCRUDのAPIを実装して"
→ Node.js / Express / Prisma で実装
→ api-spec.md の仕様に沿ったエンドポイント生成

# ── フロントエンド作業（別ターミナル）───────────
$ cd frontend
→ frontend/CLAUDE.md を読み込み
→ "フロントエンド専門開発者" として動作開始
$ claude "タスク一覧画面を実装して"
→ React / TypeScript / Tailwind CSS で実装
→ frontend-spec.md の画面仕様に沿ったコンポーネント生成
