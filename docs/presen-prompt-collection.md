# プレゼン資料作成プロンプト集（洗練版）

> **使い方**: `[テーマ]` `[対象者]` などの括弧部分を、実際の内容に置き換えてください。
> 各プロンプトは単独でも使えますが、**1 → 2 → 3 → 4 → 5** の順に組み合わせると、
> 企画から完成まで一貫したクオリティの資料が仕上がります。

---

## 1. プレゼン全体設計（ブループリント）

```
あなたはプレゼンテーション設計の専門コンサルタントです。
以下の条件で、プレゼン資料の全体設計書（ブループリント）を作成してください。

■ テーマ: [テーマ]
■ 対象者: [対象者（例：経営層 / 新規顧客 / 社内エンジニア）]
■ 発表時間: [目安の時間]

【出力に含めるもの】
1. プレゼンのゴール（聞き手にどんな行動・理解を促すか）
2. コアメッセージ（一文で言い切れるキーメッセージ）
3. スライド構成案（枚数・各パートの役割）
4. 全体のトーン＆ムード（フォーマル／カジュアル、データ重視／ストーリー重視 等）
5. 想定されるQ&Aポイント

論理的かつ聞き手を引き込む構成を意識してください。
```

---

## 2. スライド構成アーキテクト

```
以下のテーマについて、スライド単位の構成設計を行ってください。

■ テーマ: [テーマ]
■ 想定スライド枚数: [枚数、または「自動で最適化」]

【各スライドに対して出力する項目】
- スライド番号
- タイトル（簡潔で内容が一目でわかるもの）
- そのスライドの目的（「なぜこのスライドが必要か」を一文で）
- 前後スライドとのつながり（流れの接続詞的な説明）

全体として「導入 → 展開 → クライマックス → 結論」の自然な流れを設計し、
聞き手が迷子にならない構成にしてください。
```

---

## 3. ストーリー型プレゼン構築

```
以下のテーマを、ストーリーテリングの手法で魅力的なプレゼンに変換してください。

■ テーマ: [テーマ]

【ナラティブ構造】
次の5幕構成で組み立ててください。
  1. フック — 聞き手の注意を一瞬で掴む問いかけ・事実・エピソード
  2. 課題提起 — 「なぜ今これが重要か」を明確にする
  3. 洞察 — データや事例から導かれる新しい視点
  4. 解決策 — 具体的な提案・アクション
  5. テイクアウェイ — 聞き手が持ち帰る「一つのメッセージ」

プロフェッショナルな信頼感を保ちながらも、
聞き手の感情に訴えかける構成にしてください。
各幕にどのスライドが対応するかも示してください。
```

## 4. ビジュアル＆デザインディレクション

```
以下のプレゼン資料に対して、スライドごとのビジュアル・デザイン方針を提案してください。

■ テーマ: [テーマ]
■ トーン: [例：洗練・信頼感 / 活気・革新的 / 落ち着き・知性的]

【各スライドに対して提案する項目】
- レイアウトパターン（2カラム / 大数字コールアウト / グリッド配置 / 画像ブリード 等）
- 推奨する図表・チャートの種類（棒グラフ、フロー図、比較表 等）
- アイコン・イラストの活用方針
- 色使いのポイント（アクセントカラーの配置、コントラスト確保）

【全体デザイン方針も含めてください】
- カラーパレット（プライマリ / セカンダリ / アクセント の3色、HEXコード付き）
- フォントペアリング（見出し用 / 本文用）
- 視覚的モチーフ（資料全体で統一する装飾要素）

「テキストだけのスライドは作らない」を原則として、
クリーンかつモダンな印象に仕上がるよう方針を立ててください。
```

## 5. スライドコンテンツ生成

```
以下の条件で、各スライドの本番用コンテンツを作成してください。

■ テーマ: [テーマ]
■ 対象者: [対象者]
■ スライド構成: [構成案をここに貼る、または「最適な構成を提案して」]

【各スライドの出力フォーマット】
━━━━━━━━━━━━━━━━
スライド [番号]: [タイトル]
━━━━━━━━━━━━━━━━
● メインポイント（箇条書き3〜5項目、各1〜2行で簡潔に）
● スピーカーノート（口頭で補足する内容、2〜3文）
● ビジュアル要素の指示（このスライドに配置すべき図表・画像の説明）
━━━━━━━━━━━━━━━━

【守るべきルール】
- 1スライド＝1メッセージの原則
- 箇条書きは体言止め or 動詞終止で統一
- 専門用語には必要に応じて括弧書きで補足
- 聞き手がスライドを見て3秒で要点を把握できる情報量に収める
```

---

## 6. スライド簡潔化＆明瞭化エディター

```
あなたはプレゼン資料専門の編集者です。
以下のコンテンツを「スライド向け」に最適化してください。

■ 対象コンテンツ:
[ここに既存のスライド原稿・文章・メモなどを貼り付け]

【編集の方針】
1. 情報の圧縮 — 冗長な説明を削り、各スライドの文字量を最小限にする
2. メッセージの研磨 — 曖昧な表現を具体的・断定的な言い切りに変える
3. 1スライド1メッセージの徹底 — 複数の論点が混在するスライドは分割を提案
4. 視認性の確保 — 長文の段落を箇条書き・キーワード・数字に変換
5. 構造の整理 — 論理の飛躍や重複がないか確認し、流れを整える

【出力フォーマット】
各スライドについて以下を出力してください。
━━━━━━━━━━━━━━━━
スライド [番号]: [タイトル]
━━━━━━━━━━━━━━━━
▼ 変更前の問題点（1〜2文で指摘）
▼ 変更後のコンテンツ（最適化済みの本番テキスト）
▼ 編集メモ（なぜこう変えたかの簡潔な理由）
━━━━━━━━━━━━━━━━

【守るべき基準】
- 箇条書きは1項目あたり最大2行（それ以上は分割 or 削除）
- 聞き手が「読む」のではなく「見てわかる」状態を目指す
- 元の意図や専門性は損なわず、表現だけを磨く
```

## 💡 組み合わせのコツ

| ステップ | プロンプト | 得られるもの |
|---------|-----------|------------|
| ① 企画 | 1. 全体設計 | ゴール・メッセージ・方向性 |
| ② 骨格 | 2. 構成設計 | スライドの流れと各スライドの役割 |
| ③ 物語 | 3. ストーリー構築 | 感情に訴える語りの構造 |
| ④ 見た目 | 4. デザイン方針 | カラー・レイアウト・フォントの指針 |
| ⑤ 中身 | 5. コンテンツ生成 | 各スライドの完成テキスト |
| ⑥ 推敲 | 6. 簡潔化＆明瞭化 | 情報圧縮・表現の研磨 |

前のステップの出力を次のプロンプトに渡すことで、一貫性のある資料が効率よく完成します。
プロンプト6は、既存資料のリライトにも単独で活用できます。
