xAI Enterprise API

xAI Enterprise APIは、高性能なRESTfulインターフェースを提供し、 既存システムへのシームレスな統合を実現します。 OpenAI REST APIとの完全な互換性を持ち、簡単に移行が可能です。

https://api.x.ai
API Version
v1
認証方式
Bearer Token

30日間

APIキー認証

Authorization: Bearer <API_KEY>

すべてのAPIリクエストにはAuthorizationヘッダーが必要です。 APIキーはxAI Consoleで作成でき、通常「xai-」で始まります。 環境変数または.envファイルに保存することを推奨します。

# 環境変数の設定 export XAI_API_KEY="xai-your-api-key-here" # curlでのリクエスト例 curl https://api.x.ai/v1/chat/completions \ -H "Authorization: Bearer $XAI_API_KEY" \ -H "Content-Type: application/json"
💡 APIキーは安全に保管し、公開リポジトリにコミットしないでください。

🚀 Responses API（推奨）
💬

Responses API

POST /v1/responses

xAIモデルとのインタラクションに推奨される方法です。 ステートフルな会話が可能で、以前のプロンプト、推論内容、モデルの応答がxAIサーバーに保存されます。 response_idを使用して会話を継続できます（30日間有効）。

// Responses APIの基本リクエスト { "model": "grok-4", "messages": [ { "role": "user", "content": "こんにちは、Grok!" } ], "stream": false, "store": true // デフォルトtrue }
📝 主要パラメータ
model string 必須
使用するモデルID（例: grok-4, grok-3-beta）
messages array 必須
メッセージオブジェクトの配列（role: user/assistant/system, content: string）
stream boolean オプション
ストリーミングレスポンスを有効化（デフォルト: false）
temperature number オプション
出力のランダム性を制御（0-2、デフォルト: 1）
max_tokens integer オプション
生成する最大トークン数
store boolean オプション
会話をサーバーに保存（デフォルト: true）。falseで無効化
previous_response_id string オプション
以前のresponse_idを指定して会話を継続
💡 Chat Completions APIとの主な違い：Responses APIではresponse_idを使用して会話を継続でき、全メッセージを再送信する必要がありません。
💬 Chat Completions API（非推奨）

Chat Completions API

POST /v1/chat/completions

OpenAI互換のChat Completions APIです。ステートレスで、各リクエストに全会話履歴を含める必要があります。 新規プロジェクトではResponses APIの使用を推奨します。

// Chat Completions APIリクエスト { "model": "grok-3-beta", "messages": [ { "role": "system", "content": "あなたは親切なアシスタントです。" }, { "role": "user", "content": "東京の天気について教えて" } ], "temperature": 0.7 }
⚠ Anthropic SDK互換機能は完全に廃止されました。Responses APIまたはgRPCへの移行を推奨します。
🔧 サーバーサイドツール
🧰

Agent Tools API

Responses API内で利用可能

Responses APIでは、モデルが推論プロセス中に自律的に実行できるサーバーサイドツールを提供しています。 これにより、リアルタイム情報の取得やコード実行が可能になります。

  • web_search - リアルタイムのWeb検索とページ閲覧。Live Searchではストリーミングレスポンスで引用が含まれます。
  • x_search - Xの投稿、ユーザー、スレッドの検索。
  • code_execution - 安全なサンドボックス環境でPythonコードを実行。計算やデータ分析に最適。
  • document_search - アップロードされたファイルからインテリジェントな文書検索・取得（引用付き）。
  • MCP接続 - 外部MCPサーバーに接続してカスタムサードパーティツールを利用。
// ツールを有効にしたリクエスト { "model": "grok-4.1-fast-reasoning", "messages": [...], "tools": [ { "type": "web_search" }, { "type": "code_execution" } ] }
⚡ ストリーミング
🔃

リアルタイムストリーミング

stream: true

ストリーミングを有効にすると、レスポンスがトークンごとにリアルタイムで送信されます。 インタラクティブなアプリケーションに最適で、ユーザー体験を向上させます。 Live Searchを使用する場合、引用はストリーム完了時に含まれます。

# Python SDKでのストリーミング例 from xai_sdk import Client client = Client() for chunk in client.chat.completions.create( model="grok-4", messages=[{"role": "user", "content": "Hello"}], stream=True ): print(chunk.choices[0].delta.content, end="")
📦 SDK互換性

SDK & ライブラリ

pip install xai-sdk

xAI APIはOpenAI SDKと完全互換です。既存のコードをわずかな変更で移行できます。 また、ネイティブのxAI Python SDKも提供しています。

xAI Python SDK
ネイティブSDK。pip install xai-sdkでインストール。全機能に対応。
OpenAI SDK互換
base_urlを変更するだけで移行可能。既存コードをそのまま活用。
gRPC
高性能が必要な場合に最適。低レイテンシーでの通信が可能。
# OpenAI SDKからの移行例 from openai import OpenAI client = OpenAI( api_key="xai-your-api-key", base_url="https://api.x.ai/v1" # これだけ変更 ) response = client.chat.completions.create( model="grok-4", messages=[{"role": "user", "content": "Hello"}] )
