マルチエージェントAI の基本概念

なぜ複数のAIエージェントを協力させると強いのか。
オーケストレーター・サブエージェントの仕組みと主要な協調パターンを理解する。

マルチエージェントAIとは？

定義

マルチエージェントシステムとは、それぞれ異なる役割・指示・ツールアクセスを持つ複数のAIエージェントが連携して、単一エージェントでは困難な複雑タスクを達成するシステムです。GitHub を共通のプラットフォームとして使うことで、Issue → コード生成 → テスト → レビュー → PR マージまでを自動化できます。

単一エージェント

  • コンテキストウィンドウが1つに限定
  • 長大タスクで精度が低下
  • 並列処理ができない
  • 失敗時に全体がブロック
  • 役割の切り替えでミス発生
VS

複数エージェント

  • 各エージェントが独立したコンテキスト
  • 並列実行で大幅な高速化
  • 役割特化でハルシネーション減少
  • 障害が他エージェントに波及しない
  • 専門性を最大活用できる
コアコンセプト
🎯

オーケストレーター

全体を俯瞰しタスクを分解・割り当て・結果を統合する司令塔エージェント。他エージェントの出力を評価し次のアクションを決定する。

⚙️

サブエージェント

特定の専門タスクを実行する実行者エージェント。固有のシステムプロンプト・ツールアクセスを持ち、独立したコンテキストで動作する。

🔧

ツール (Tool Use)

エージェントが呼び出せる外部機能。ファイル読み書き・コード実行・Web検索・GitHub API・外部サービスなど。MCPで拡張可能。

📋

コンテキスト管理

各エージェントは独立したコンテキストウィンドウを持つ。共有メモリ・アーティファクト（spec.md等）を介して情報を受け渡す。

🛡️

ガードレール

エージェントの行動を制御する制約・検証ルール。スコープ逸脱・破壊的操作・セキュリティリスクを防ぐ安全機構。

🔗

MCP (Model Context Protocol)

AIエージェントが外部ツール・データソースと接続する標準プロトコル。GitHub・Slack・DB等と簡単に連携できる。

GitHub をエージェント協調の基盤にする理由

Issue がタスクキュー

GitHub Issues をエージェントへのタスク割り当てに使用。Issueを作るだけでエージェントが自律的に作業を開始できる。ラベルで優先度・エージェント種別を制御。

PR が成果物レビューの場

エージェントが作成した PR をレビューエージェント・人間が確認。インラインコメントへの対応もエージェントが自動実行。68.6%の自動マージ成功率。

Actions が実行環境

GitHub Actions がエージェントの安全な実行環境を提供。CI/CD パイプラインにエージェントを組み込み、テスト・ビルド・デプロイを自動化。

MCP でツール拡張

GitHub の MCP サーバーでリポジトリ検索・Issue管理・PR作成をエージェントが直接実行。外部サービスとの連携も MCP プロトコルで標準化。

重要な設計原則

マルチエージェントシステムの失敗の多くは、モデルの能力ではなく「構造の欠如」が原因です。成功の鍵は3つ：明確なタスク境界・構造化されたハンドオフアーティファクト・明示的なガードレール。技術よりも設計と役割定義に注力することが最重要です。
