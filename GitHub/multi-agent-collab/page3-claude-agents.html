Claude Code — ターミナルネイティブのエージェント

Claude Code とは

Claude Code は開発環境内で直接動作するターミナルファーストのエージェント型コーディングツールです。コードを書くだけでなく、ファイル操作・シェル実行・Web検索・MCP連携まで行う自律エージェントとして動作します。

  • ファイルの読み書き・ディレクトリ操作
  • シェルコマンドの実行（テスト・ビルド・デプロイ）
  • Web 検索でリアルタイム情報を取得
  • MCP サーバーで外部ツールと連携
  • Computer Use でデスクトップを自律操作（クリック・入力・スクリーンショット）
  • Remote Control (2026/02): モバイルから HTTPS 経由でローカルセッションを操作

CLAUDE.md — プロジェクトレベルの指示ファイル

CLAUDE.md はリポジトリに配置するエージェントへの永続的な指示ファイルです。コーディング標準・アーキテクチャ原則・役割定義・禁止事項を記述することで、すべてのセッションで一貫した動作を実現します。

# CLAUDE.md — プロジェクト指示例 ## あなたの役割 あなたはシニアバックエンドエンジニアです。 TypeScript / Node.js を使用します。 ## コーディング規約 - 関数は単一責任原則に従う - テストは必ず書く（カバレッジ 80% 以上） - コミットメッセージは conventional commits 形式 ## 禁止事項 - main ブランチへの直接プッシュ禁止 - 未テストのコードをコミットしない
Claude Agent SDK — エージェントシステムの構築基盤

Agent SDK とは

Claude Agent SDK (旧 Claude Code SDK) は、Claude Code を動かしているエージェントループ・ツール・コンテキスト管理を汎用ライブラリとして提供します。Python / TypeScript 対応で、カスタムのマルチエージェントシステムを構築できます。

  • Python v0.1.48 / TypeScript v0.2.71 (2026年初頭時点)
  • ツール・フック・MCP・サブエージェント・オーケストレーションパターンを内包
  • Semantic Kernel (Microsoft) との統合に対応

オーケストレーター / サブエージェント パターン
🎯

オーケストレーター（プロジェクトリード）

全体を俯瞰しタスクを分解。フィーチャーリードエージェントに委託。全体の品質と一貫性を管理。独立したコンテキストウィンドウを持つ。

↓ 委託
📋

フィーチャーリードエージェント（中間層）

機能単位のタスクを管理。専門エージェントへ具体的な作業を委託。上位層からは結果のみを返す（コンテキストを汚染しない）。

↓ 委託
⚙️

スペシャリストエージェント（実行層）

Data・Logic・API・Test の各専門エージェントが独立した環境で実行。各エージェントは専門ツールのみにアクセス。並列実行で高速化。

Agent Teams — 並列協調（実験的機能）

チームリードとチームメイト

1つのセッションがチームリードとして共有タスクリストを管理。チームメイトが並列コンテキストウィンドウで同時作業。

  • 複数の Claude インスタンスが同時に異なるサブタスクを実行
  • コミュニケーションはダイレクト
  • コンフリクトを自動的に解決
  • 変更を継続的にマージ

実装フレームワーク例

コミュニティで人気のマルチエージェントフレームワーク:

  • Metaswarm: 18エージェント・13スキル・15コマンド・TDD強制・品質ゲート搭載
  • ccswarm: Git ワークツリーでエージェントを分離し安全な並列開発
  • Claude Agent SDK: 公式 SDK でカスタムシステムを構築

Claude Code と GitHub Copilot の使い分け

GitHub Copilot はGitHubのエコシステム内でネイティブに動作し、Issue/PR/Actionsとの統合が深い。Claude Code は開発環境のターミナルやMCPで高度なカスタマイズが可能。両者を組み合わせることで、GitHub上の自動化（Copilot）と複雑なコーディング・設計タスク（Claude Code）を使い分ける構成が最も効果的です。
