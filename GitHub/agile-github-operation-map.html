5 用語集
プロダクトバックログ — 実現したい機能・改善を優先順位付きで管理する一覧
スプリント — 1〜4週間の固定タイムボックスで開発サイクルを回す単位
ユーザーストーリー — 「〜として〜したい」形式で記述する要求の最小単位
インクリメント — スプリント毎に生まれる動作する成果物
カンバン — タスクの流れを可視化し、ボトルネックを発見するボード
WIP制限 — 同時進行タスク数の上限。流れの最適化に不可欠
ベロシティ — 1スプリントで完了するストーリーポイントの合計値
Definition of Done — 「完了」とみなす基準の合意（テスト済・レビュー済等）
Issue — GitHubでタスク・バグ・要望を管理するチケット
Pull Request — コード変更をレビュー・議論してマージする仕組み
6 本質のまとめ
Agile = 意思決定の循環
Backlog = 未来の価値の優先順位
GitHub = 実行エンジン
