カスタム指示（Custom Instructions）

📄 リポジトリ共通指示

.github/copilot-instructions.md

リポジトリ内のすべての Copilot リクエストに適用されます。プロジェクトの概要・ビルド/テスト/リントコマンド・ コーディング規約・命名規則などを記載してください。 すべての Copilot プランで利用可能です。

# プロジェクト概要
このプロジェクトは TypeScript + React 製のダッシュボードアプリです。

# 開発コマンド
- ビルド: npm run build
- テスト: npm test -- --coverage
- リント: npm run lint

# 規約
- コンポーネント名: PascalCase
- 関数名: camelCase
- 型は必ず明示的に定義すること

🗂️ パス別指示ファイル

.github/instructions/*.instructions.md

特定のファイルパスやディレクトリに対してのみ適用される指示ファイル。 YAML フロントマターで適用対象を指定します。 excludeAgent: "coding-agent" を指定すると Coding Agent からは除外されます。

---
applyTo: "app/models/**/*.rb"
---

# Rails モデル規約
- バリデーションは必ず書くこと
- スコープは named scope で定義すること
- N+1 問題に注意し includes を適切に使用すること

環境セットアップ（copilot-setup-steps.yml）

⚙️ 事前環境構築ワークフロー

.github/workflows/copilot-setup-steps.yml

Copilot がタスクを開始する前に実行されるセットアップステップを定義する GitHub Actions ワークフローファイルです。 依存関係のインストールや言語ランタイムのセットアップに使用します。 ファイルへの変更は Actions タブから通常のワークフローとして検証できます。

name: Copilot Setup Steps
on: workflow_dispatch

jobs:
  copilot-setup-steps:
    runs-on: ubuntu-latest
    steps:
      # Node.js のセットアップ
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      # 依存関係のインストール
      - name: Install dependencies
        run: npm ci

Custom Agents（カスタムエージェント）

特定タスクに特化した Copilot の専門家バージョンを作成できます。 Markdown ファイルとして .github/agents/ ディレクトリに保存します。 最大 30,000文字のプロンプトで動作を定義可能です。

YAML フロントマター（設定）

---
name: "frontend-specialist"
description: "Tailwind CSS と TypeScript を使ったフロントエンド専門家"
tools: ["read_file", "write_file", "run_command"]
mcp-servers:
  playwright:
    command: "npx"
    args: ["@playwright/mcp@latest"]
---

Markdown 本文（エージェントの指示・ペルソナ）

あなたはフロントエンドエンジニアの専門家です。

# 専門スキル
- React + TypeScript でのコンポーネント設計
- Tailwind CSS を使ったレスポンシブデザイン
- Playwright を使った E2E テスト

# 作業規約
- コンポーネントは必ず単一責任の原則を守ること
- カラーはデザイントークン変数のみ使用すること

保存場所: リポジトリ個別 → .github/agents/ ／ 組織共通 → {org}/.github/agents/ または {org}/.github-private/agents/

Agent Skills と Hooks

🎯 Agent Skills

SKILL.md + 関連ファイル

Copilot が特定タスクで参照する指示・スクリプト・リソースのフォルダ。 関連タスクが来たときに自動で読み込まれ、専門知識として活用されます。

  • SKILL.md に手順・ノウハウを記述
  • 補助的な Markdown やスクリプトを参照
  • リポジトリ内またはホームディレクトリに配置
  • 適切なタスクで自動的にロード

🪝 Hooks

エージェント実行イベント時

エージェント実行の特定ポイントでカスタムシェルコマンドを実行できる仕組み。 バリデーション・ログ・セキュリティスキャン・ワークフロー自動化に活用します。

  • 実行開始時フック
  • コミット前フック
  • ツール呼び出し後フック
  • カスタムバリデーション追加
