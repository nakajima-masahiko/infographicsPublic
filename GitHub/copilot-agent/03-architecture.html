⚙️

Tool-Use 実行ループ

Copilot は LLM の Tool-Use（関数呼び出し）機能を使い、以下のツール群を自律的に組み合わせながらタスクを遂行します。

📂

ファイル操作

ファイルの読み込み・作成・編集・削除。ディレクトリ構造の探索。

🖥️

ターミナル実行

テスト・リント・ビルドコマンドを実行し、出力を確認して次の行動を決定。

🐙

GitHub API（MCP経由）

Issues・PR・コメントの読み取り。コミットのプッシュ。

🌐

Web アクセス（Playwright）

ローカルホストのプレビューを閲覧・操作・スクリーンショット撮影。

🔄

エラー検出→自動修正

テスト失敗・リントエラーを検知し、原因を分析して自動的に修正を試みる。

📋

チェックリスト管理

タスクを項目分解し、完了のたびにPR説明欄のチェックリストを更新。

☁️

エフェメラル環境の仕様

動力源 GitHub Actions（ジョブとして実行）
ライフサイクル タスクごとに新規作成、完了後に破棄（エフェメラル）
ブランチ命名 copilot/[task-description] または copilot/issue-[番号]
PR の種類 最初はドラフト PR、完了後に正式 PR としてオープン
コスト（Actions） GitHub Actions 使用分（月次 Actions minutes 枠から消費）
コスト（AI） 1セッション = 1 プレミアムリクエスト（2025年7月〜）
セッションログ 各コミットにセッションログへの URL が含まれ、詳細な実行記録を参照可能
ネットワーク デフォルトで Firewall 有効（許可リストで管理）
🖥️

ランナーオプション

GitHub ホステッドランナー

デフォルト

GitHub が管理するクラウドランナー。設定不要ですぐに使用可能。 Firewall による保護が自動的に適用される。

セルフホステッドランナー

ARC スケールセット（2025年10月〜）

Actions Runner Controller（ARC）を使用。内部ネットワークへのアクセスが必要な場合に有効。 Firewall との互換性なし（自動無効化）。Windows は非対応。
