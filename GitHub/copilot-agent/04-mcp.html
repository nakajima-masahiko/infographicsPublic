MCP（Model Context Protocol）とは

Model Context Protocol（MCP） は、アプリケーションが大規模言語モデル（LLM）とコンテキストを共有するための オープンスタンダードです。MCP はAIモデルをさまざまなデータソースやツールに接続する 標準化された方法を提供し、それらが効果的に連携できるようにします。

Copilot Coding Agent に MCP を組み合わせることで、エージェントの機能が大幅に拡張されます。 Copilot は外部ツール・サービスに接続しながら 「アジェンティックループ」 を実行でき、 情報収集→分析→判断→実行を自律的に繰り返します。

🐙

GitHub MCP

デフォルト組み込み

Copilot が GitHub のデータにアクセスするための公式 MCP サーバー。

  • Issues・PR の読み取り
  • リポジトリ情報の参照
  • コメント・レビューの取得
  • コード検索・参照
デフォルトでは 現在のリポジトリへの読み取り専用 アクセスに限定。 権限は専用スコープ付きトークンで管理。
🎭

Playwright MCP

デフォルト組み込み

Web ページへのアクセス・操作・スクリーンショット機能を提供。

  • ローカルプレビューの閲覧
  • ページ操作（クリック・フォーム入力）
  • スクリーンショット撮影
  • UI 確認・テスト支援
デフォルトでは localhost / 127.0.0.1 のみアクセス可能。 外部 Web へのアクセスは Firewall の Allow List に追加が必要。
⚙️

カスタム MCP サーバーの設定

リポジトリの Settings → Copilot → Coding agent → MCP configuration に JSON を記入して設定します。 設定後、ツールはタスク実行中に自動的に利用可能になり、Copilot は承認なしに自律的にツールを呼び出します。

{
  "mcpServers": {
    // Slack MCP サーバーの例
    "slack": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-slack"],
      "env": {
        "SLACK_TOKEN": "$COPILOT_MCP_SLACK_TOKEN"
      }
    },
    // Neon データベース MCP サーバーの例
    "neon": {
      "command": "npx",
      "args": ["-y", "@neondatabase/mcp-server-neon"],
      "env": {
        "NEON_API_KEY": "$COPILOT_MCP_NEON_API_KEY"
      }
    }
  }
}

※ 環境変数は COPILOT_MCP_ プレフィックスのシークレットのみ MCP 設定内で使用可能

MCP 対応機能と制約

機能 対応状況 補足
MCP ツール（Tools） 対応 メインの機能。関数呼び出しとして使用
MCP リソース（Resources） 非対応 データソース参照機能は未サポート
MCP プロンプト（Prompts） 非対応 プロンプトテンプレート機能は未サポート
OAuth 認証リモートサーバー 非対応 OAuth を使うリモート MCP は利用不可
API Key 認証リモートサーバー 対応 Firewall の Allow List への追加が必要
ローカル（stdio）MCP サーバー 対応 npx / uvx での起動に対応
Custom Agent 専用 MCP 対応 そのエージェントにのみ適用される

  • 🗄️Neon Migration Specialist — Neon Postgres データベースのマイグレーション支援
