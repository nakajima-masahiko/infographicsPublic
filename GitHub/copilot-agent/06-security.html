セキュリティ設計の 4 本柱

Firewall による通信制限

デフォルトで外部通信を遮断。npm・pip 等のコアサービスのみ許可。データ漏洩リスクを最小化。

🔍

コード・シークレットスキャン

Secret Protection・Code Security・Supply Chain Security を実行。PR 確定前に脆弱性を検出・修正。

✅

強制的な人間レビュー

Copilot は自分の PR を承認・マージできない。依頼者本人も承認不可。必ず独立したレビューが必要。

📋

完全な監査ログ

全コミットにエージェントセッションログへのリンクを付与。すべての変更履歴を追跡・監査可能。

🔥

Firewall 詳細設定

🛡️

デフォルト Firewall（ON）

Copilot Coding Agent はデフォルトで Firewall が有効化されており、外部ネットワークへのアクセスが制限されます。 これにより意図しないデータ送信や外部サービスへの不正アクセスを防ぎます。

  • デフォルト許可: npm・pip・Docker Hub・GitHub.com 等のコアサービス
  • カスタム許可: リポジトリ Settings → Copilot → Coding agent の Allow List で追加
  • セルフホステッドランナー使用時は自動的に Firewall が無効化される
  • Windows ランナーは Firewall 非対応
  • Firewall を完全無効化することも可能（セキュリティリスクに注意）
🔐

PR 承認・マージのルール

  • 🚫

    Copilot は自分の PR を承認・マージできない

    Copilot が作成した PR は、必ず別の人間の開発者がレビューし承認する必要があります。

  • 🚫

    タスク依頼者本人は承認できない

    Copilot にタスクを割り当てた開発者は、そのPRの承認者になれません。独立したレビューが保証されます。

  • ⚠️

    CI/CD は書き込み権限者の承認後に実行

    Copilot が作成した PR の CI/CD ワークフローは、書き込み権限を持つユーザーが承認するまで自動実行されません。権限昇格攻撃を防ぐセキュリティ機能です。

  • ✅

    既存のブランチ保護ルールがそのまま適用される

    リポジトリに設定した Required Reviews・Status Checks 等のブランチ保護ルールはすべて Copilot の PR にも適用されます。

  • ✅

    組織のポリシーが自動的に尊重される

    Organization レベルで設定したポリシーやコードオーナー設定は自動的に反映されます。

⚠️

利用制限（Limitations）

📁

単一リポジトリのみ変更可能

1タスクで変更できるのは割り当て時に指定した1つのリポジトリのみ。複数リポジトリの横断変更は不可。

🔀

PR は1タスクにつき1件のみ

1つのタスクに対して Copilot が開く PR は必ず1件。複数 PR を同時作成することはできない。

💬

割り当て後のIssueコメントは無視

タスク割り当て後にIssueへ追加したコメントはCopilotに届かない。フィードバックはPRコメントで行う。

🔒

自分が作成していないPRは変更不可

Copilot は自分が作成した PR のみ修正可能。他のPRへの変更は行えない。

🔌

OAuth 認証 MCP は非対応

OAuth を使うリモート MCP サーバーには接続できない。API Key 認証方式を使用すること。

🏢

GitHub ホステッドランナーが必要（デフォルト）

org 全体でホステッドランナーが無効化されている場合、セルフホステッドランナーへの切り替えが必要。

💰
