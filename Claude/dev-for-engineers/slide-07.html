ユースケース 04 & 05 — Docs & Tests

ドキュメント・テスト自動化

「後でやろう」が積み重なる2大タスクを、実装と同時に自動化する。
コードを渡して「ドキュメントを書いて」「テストを書いて」と伝えるだけ。

📄 ドキュメント生成
README 自動生成
プロジェクト概要・セットアップ・使い方を構造化して生成
JSDoc / TSDoc コメント
関数・クラスへの型付きドキュメントコメントを一括追加
API仕様書（OpenAPI形式）
コードからSwagger互換のYAMLを自動生成
「この関数にJSDocコメントを
追加してください：[コード]」
🧪 テストコード生成
ユニットテスト生成
Jest / Vitest / pytest などのフレームワークに合わせた正常系テスト
エッジケース発見
「見落としているエッジケース」を指摘・テストケースとして追加
モック・スタブ生成
外部API・DBのモックを適切な形で自動生成
