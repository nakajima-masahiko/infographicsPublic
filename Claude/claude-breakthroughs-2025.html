Computer Use — AIが初めてPCを操作する

Claude 3.5 Sonnet が「Computer Use」機能を公開。スクリーンを見て、マウスを動かし、キーボードを打ち、ファイルを操作する——まるで人間のようにPCを自律的に操作できるAIが世界で初めて商業利用可能になった瞬間。 ブラウザでの情報収集・フォーム記入・コードの実行まで、複雑なワークフローを完全自動化。

2024年 11月 オープン標準 MCP

Model Context Protocol (MCP) — AIツール接続の共通言語

Anthropic が MCP をオープンソースとして公開。AIとデータソース・ツールを接続する標準プロトコルを策定。 GitHub・Slack・Google Drive・データベース・社内APIなど、あらゆるシステムを同一のインターフェースでClaudeに接続できる。 これにより開発者は個別の統合を作る必要がなくなり、AIエコシステムが爆発的に拡張。

採用企業 Block, Apollo, Zed, Replit コネクタ GitHub / Slack / Postgres / etc. プロトコル JSON-RPC ベース

Computer Use in Action

Computer Use の実際の動き

Claude がどのようにPCを操作するか、実際の処理フローを見てみよう。

1
スクリーンショット取得
現在の画面を画像として受け取り、UI要素（ボタン・テキスト・アイコン）を視覚的に認識。
2
状況理解と計画
「何をするべきか」を目標から逆算してアクションプランを生成。Extended Thinking と連携。
3
アクション実行
座標指定クリック・キーボード入力・スクロール・ドラッグを API 経由で実行。
4
結果確認 & 修正
実行後の画面を再取得し、成功・失敗を判断。失敗した場合は別アプローチを試みる。
5
タスク完了報告
最終的な結果をユーザーに自然言語で報告し、スクリーンショットを証拠として提示。

Claude Code Workflow

Claude Code — ソフトウェア開発の自動化フロー

"Claude Code はターミナルから直接動く自律型エージェントです。コードベースを理解し、 バグを修正し、機能を追加し、テストを書き、PRを作成する——すべてあなたの自然言語指示だけで。"

— Anthropic, Claude Code リリースノート (2025)

標準的な開発ワークフロー

1
コードベース全体を把握
リポジトリ構造・依存関係・テスト状況を自動スキャン。CLAUDE.md の指示を読み込む。
2
Plan モードで戦略立案
変更前に実装計画を提示。影響範囲・リスク・テスト方針を確認してから実行。
3
コード修正 + テスト自動実行
ファイル編集後、自動でテストスイートを実行。失敗したら自律的に修正を繰り返す。
4
Git コミット & PR 作成
コミットメッセージを生成しコミット。GitHub PR も説明文付きで自動作成。
